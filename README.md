# AKI Risk Suite

**Інструменти стратифікації ризику гострого ураження нирок (ГУН)**

Інтерактивний вебдодаток для клінічної оцінки ризику ГУН за чотирма валідованими прогностичними шкалами. Уся логіка виконується локально у браузері — без сервера, без мережевих запитів, без збереження даних пацієнта.

> ⚠️ **Інструмент підтримки клінічних рішень для медичних працівників. Не є засобом діагностики чи лікування.** Див. розділ [«Дисклеймер»](#дисклеймер).

---

## Калькулятори

| Шкала | Призначення | Дискримінація | Першоджерело |
|---|---|---|---|
| **Thakar** | ГУН після кардіохірургії (CSA-AKI); прогноз потреби в діалізі | C-статистика ≈ 0.76–0.82 | Thakar CV та ін., *J Am Soc Nephrol* 2005 |
| **Mehran 2.0** | Контраст-індукована ГУН (CI-AKI) після ПКВ / коронарографії; доконтрастна модель 1 | AUROC ≈ 0.84 | Mehran R та ін., *Lancet* 2021 |
| **AKICS** | ГУН будь-якої стадії після кардіохірургії; інтегрує перед-, інтра- та післяопераційні фактори | AUROC ≈ 0.84 | Palomba H та ін., *Kidney Int* 2007 |
| **Хірургічний індекс ГУН** | ГУН після некардіохірургічних (загальнохірургічних) втручань; класи I–V | C-статистика ≈ 0.80 | Kheterpal S та ін., *Anesthesiology* 2009 |

Кожна шкала реалізована за автентичною точковою системою першоджерела з відповідними категоріями ризику та прогнозованими частотами ГУН.

## Можливості

- **Реактивний розрахунок** — бал та категорія ризику оновлюються миттєво під час введення даних, без кнопки «Розрахувати».
- **Спільні параметри** — вік, стать, креатинін / ШКФ, супутні патології та глюкоза синхронізуються між калькуляторами.
- **Конвертація одиниць** — миттєвий перемикач креатиніну мг/дл ⇄ мкмоль/л.
- **Розрахунок ШКФ** — за формулою CKD-EPI 2021 (без расового коефіцієнта) з можливістю ручного введення.
- **Блок доказовості** — для кожної шкали наведено показник валідації, першоджерело та доказові клінічні рекомендації відповідно до рівня ризику.
- **Експорт зведення** — копіювання структурованого підсумку оцінки у буфер обміну.
- **Адаптивний інтерфейс** — двоколонковий дашборд на десктопі, одноколонковий на мобільних пристроях.

## Технічні характеристики

- Один автономний файл `index.html` — HTML, CSS (Tailwind через CDN) та чистий JavaScript (Vanilla JS).
- Без бекенду та зовнішніх залежностей для розрахунків.
- Працює офлайн після першого завантаження сторінки.
- Введені дані не зберігаються та не передаються.

## Розгортання

Додаток — статичний файл, тож його достатньо розмістити на будь-якому статичному хостингу.

**GitHub Pages:**
1. Завантажте `index.html` у репозиторій (файл має називатися саме так).
2. *Settings → Pages → Build and deployment*: Source — *Deploy from a branch*, Branch — `main`, папка `/ (root)`.
3. За ~1 хвилину сайт буде доступний за адресою `https://<користувач>.github.io/<репозиторій>/`.

**Локально:** завантажте `index.html` та відкрийте його у браузері — інтернет не потрібен.

## Дисклеймер

Цей застосунок призначений виключно для кваліфікованих медичних працівників як **інструмент підтримки клінічних рішень**.

- Інструмент **не є засобом діагностики чи лікування** і не призначений для пацієнтів.
- Результати — це **оцінки ймовірності** на основі опублікованих прогностичних моделей. Вони **не замінюють** клінічну оцінку, обстеження пацієнта та інституційні протоколи.
- Прогностичні моделі мають **обмеження валідації**: шкала AKICS не проходила зовнішньої валідації; Mehran 2.0 реалізовано як доконтрастну модель 1 (8 змінних); усі шкали валідовані в окремих популяціях і можуть бути неточними поза ними.
- **Остаточне рішення** щодо ведення пацієнта завжди приймає лікар.
- Перед впровадженням у закладі охорони здоров'я інструмент має бути погоджений із **клінічним врядуванням установи**.
- Автори не несуть відповідальності за клінічні рішення, прийняті з використанням цього інструмента.
